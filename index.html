Meteor

An Open-source Isomorphic JavaScript Web Framework.
It is a cohesive development platform with a collection of libraries
and packages that are bound together to build extraordinarily
dynamic web pages and mobile applications.

DDP

A Client-Server Protocol based on WebSockets for queuing and updating a Server-Side database and for synchronizing such updates among clients.

Built-In MongoDB

Meteor's built-in support of MongoDB makes it easier for applications that need a flexible architecture, cloud server support and real-time updates.

Other Databases

Meteor can be intergrated with distributed databases like RethinkDB, graph databases like Neo4j, relational databases like PostgreSQL, AlaSQL, Oracle.

Why DDP is essential for Meteor?

Distributed data protocol is essential for Meteor because it is a universal protocol which supports all programming languages, databases and frameworks, which provides solution to most common Client-Server problems. It made server implemetation for Meteor easy by simple command 'Meteor.publish' and publish it to client by 'Meteor.subscribe'.

SSR in Meteor

Server side rendering provide ultra-simple configuration and deployment options. Having server side rendering instead of client side rendering saves page load time and it is essertian in Meteor because with Meteor we can get whole stack up and run quickly along with SSR. So, this makes us concentrate just on coding.
