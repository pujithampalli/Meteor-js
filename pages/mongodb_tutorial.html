MongoDB using Meteor

This Page goes through the basic Mongo Collection & CRUD operations performed using simple application in Meteor.


Setting up the environment

To begin, first create a sample meteor application.
      • Install Meteor
      • Create sample Application using MeteorCreateSample

Once you have created the sample Application. You will see two folders Client and Server and each has main.js file which is the startup file for Server and Client. As meteor is a full-stack framework, the code that you write will run on both the server and the client.

Coming to the data storage and retrieval Meteor uses MongoDB as the default database. You can read more about Meteor MongoDB workflow in the Meteor uses MongoDB. Collections are Meteors way of storing data and creating a collection is as simple as calling

  • Tasks = new Mongo.Collection("tasks");

In your Server and Client JavaScript file adding the above line sets up a MongoDB collection “tasks” on the server side and on the client side it creates a cache of the server collection
