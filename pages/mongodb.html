Meteor uses MongoDB

Meteor follows the concept of Mini Databases. A Mini Database is a mini (lite) version of the normal database that sits in the memory of the client side. What this does is, instead of client sending requests to the server and waiting for it to update, the client can directly change the desired contents in the mini database which automatically syncs with the main server database using Distributed Data Protocol (DDP). When the client notices a change, it updates the change in the mini mongo instance first and checks again the client side and once it’s done then it lets the mini mongo handlers to communicate with the server’s main database through DDP to let server know the change. The server on the other side can accept or reject the changes made by any client. If the server accepts the change then the server sends the changed message to every client who is connected to the server. Also, if the server rejects a change the instance of mini database is corrected.




Advantages of this Structure

  • The whole process is asynchronous and it’s done using DDP which means client doesn’t have to wait for the server response, it can update the code based on the data that is available in the mini database
  • Once there is any change then the client will be updated immediately which reduce the wait time
  • Slow internet connection or lost connection doesn’t impact the client like other normal environment. Client doesn’t have to wait for the server permission to make changes. You can work with the data that is available in the mini database and once the lost connection is back, all of the data that is in the queue will be sent all together to the server and the server upon receiving will send authorized changes to the client
  • Client can take Meteor on faith most of the times the data sent to the server will be broadcasted to all of the connected clients immediately

    • By default, every Meteor project comes with a database without any external configuration or setup. For suppose if you create a project then the database shows up automatically once the local server is started. However, this database is not SQL but Mongo database.

    MongoDB is an open source database that uses a document-oriented data model. MongoDB stores data in JSON-like documents, means that fields and data structure can vary. When it comes to comparison Mongo database is bit different from normal SQL. Mongo uses different names for the most familiar concepts like tables, rows, columns and others. It refers them by collections, documents, fields. a Mongo collection is equivalent to an SQL table. Terminology is different but conceptually it’s the same.


    Create a Collection

    • NewList = new Mongo.collection(‘UniqueName’) #Creates a table with the given name and NewList is a variable

      • Note : The name for the collection should be Unique otherwise Meteor will throw an error.


      Insert Data

      Inside the console,

    • NewList.insert();

      • The above function is used to add the values to the created variable that is assigned to the collection. Here the data that we give to this function should be in JSON format like :

    • { name : “Meteor Mongo”, course: “CS6320”, section: 1 }

      • To insert this data into the “newList” collection, pass it into the above insert() function inside the console

    • NewList.insert({name : “Meteor Mongo”, course: “CS6320”, section: 1})

      • Once this function is executed then a row i.e., documents in MONGO db with the given data will be created. Also, Mongo db attaches a new unique id to each and every document. This unique ID is known as Primary Key.


      Retrieve Data

      Inside the console,

    • NewList.find();

      • Here, we’re using find function, that is used to retrieve data from our collection. It is not mandatory to pass arguments to the find function. If the args list is empty then the function displays all the documents from the collection. However, if you want to retrieve a specific items from the collection then a JSON data can be passed to the find function.


    • NewList.find({course : “CS6320”})

      • The above function retrieves the document that has course as ‘CS6320’. If there are multiple documents with the same course name then all of them will be retrieved.


      Update Data

      Inside the console,

    • NewList.update({course : “CS6320”}, { $set: { course: "6320" } })

      • Above, we can use update a single document by giving the old details and the new entry and seeting it to the old one.


      Delete Data

      Inside the console,

    • NewList.remove({id: “y485k2BoHyBy8M9EL” })

      • Above function removes the document of the matched data from the collection. You can delete the document by giving the id. Also, there are many other functions for delete for removing all the documents from the collection, multiple documents of the same given name and many other. These are the basic CRUD operations, there are many more functions available for MongoDB like all other databases.
