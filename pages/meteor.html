METEOR

Meteor was first introduced under the name Skybreak in December 2011 and it was developed by the Meteor Development Group. Later on, Meteor acquired Y Combinator in October 2014 and Kadira in March 2017.“Meteor Development Group” has built some libraries like Tracker and Blaze that is for a reactive front-end experience.

Meteor is an open-source and free JavaScript web framework written using Node.js. It is a cohesive development platform with a collection of libraries and packages that are bound together to build extraordinarily dynamic web pages and mobile applications. Meteor can be used with Blaze templating engine, the Angular framework or React library in the client side. Meteor is built using JavaScript file and it is deployed to both the server and the client. Meteor uses the Distributed Data Protocol and a publish–subscribe pattern and it is integrated with Mongo DB.

Components

• Node.js - Javascript based web server

• MongoDB - NoSQL database that "talks" JSON (javascript)

• Tracker - transparent reactive programming

• DDP - Distributed Data Protocol "REST for websockets"

• Blaze/React/Angular - Page templating technologies

• ISOBuild - a build system for application deployment

• Other community libraries - standard definition for adding 3rd party packages to Meteor applications

Advantages

• Developed in one language – Meteor is using JavaScript in all environments It is using pure JavaScript which makes it easy to switch between front- and back-end tasks (application server, web browser, and mobile device)

• data on the wire - Unlink other web framework, the server sends data (not HTML) and the client renders it.

• full stack reactivity - without a browser refresh every change made to HTML, CSS, JavaScript on the server are reflected automatically. This is called Live html. As the application’s layers from template to database gets update automatically, it is not necessary to refresh the page to see implemented updates.

• Latency compensation– When data is modified on the client, these modifications appear as if they happened on the server without any delay. Meteor can access data instantaneously without waiting for a round- trip to the server. Meteor is extremely important for creating a responsive web application.

• Built-in front-end solution-It Is not necessary to use frameworks such as Angular, Ember. But if there is need, Meteor allows to add another framework like React or Angular to it.

• Rapid prototyping–Meteor produces cross-platform code (iOS and Android App). It gives an opportunity to turn web application into a smartphone app using Cordova, for example Cordova for making some simple apps and creating prototypes for further development on the other platforms.


key features

• Communication Layer/The Server – How server interact in Meteor Js

Extension to Json is used for serializing and deserialize data moving across the wire via DDP. To reflect changes only on server If (Meteor.isServer) {} is used.

• Browser/Frontend – Features front end users to interact in Meteor Js

To reflecting changes on frontend/browser if (Meteor.isClient) {} is used.

• Methods – Meteor helps in creating a complex function easier

The method is defined in common code on client & server to enable optimistic UI. In the basic app, meteor method is simple as defining a function. In complex app, few extra features are defined to make it powerful and easily testable. The method is an API endpoint for our server, we can define a Method on server & its counterpart on the client, then call it with some data, write to the database and get the return value in a call-back.

• Publish – Fetching data from server

There can be lot of data which can’t be available to users so for that we tell meteor that publish unflagged posts only and there is no way that client can see flagged posts. This makes meteor app more secure.

• Subscribe – Using data on client side

Even though we don’t’ want a client to see flagged posts but we cannot send them all data. He should have access to particular data which he has subscribed for. Any data we subscribed will be mirrored on client thanks to Minimongo.

• Helpers – Interaction with the UI

After publishing and subscribing method now we have to deploy it on UI. So for interaction with UI, helpers method is used which provides a display. It’s a part of ReactiveContext and will available on each $scope and context and it will stop automatically when $scope is destroyed.

• Autorun – Checks data modification

Autorun establishes a reactive context and whenever any reactive data it depends on modify then it is re-executed. It always runs at the time of first execution. So each autorun should be limited to a particular task. For example, you have an autorun that transforms data when a home button is clicked, and another that retrieves a new page data when URL parameters are updated. So, these tasks are not related to each other so both should have separate autoruns assigned to them, otherwise both will run every time either scenario is triggered.
